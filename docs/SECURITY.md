# 安全设计

## 威胁模型
数据库、服务器管理员与备份读取者看不到条目明文；浏览器解锁期间的 XSS、恶意扩展、键盘记录器及弱主密码不在保护范围内。

- PBKDF2-SHA-256 310,000 次；每用户随机盐。
- AES-256-GCM，所有加密使用独立 96-bit IV。
- 服务端认证与加密密码相同但用途分离：服务端仅得到登录密码并以 scrypt 哈希；客户端 KEK 不上传。
- HttpOnly、Secure、SameSite=Strict 会话；同源+CSRF 双校验；登录限速。
- CSP 禁止第三方脚本及内联脚本。

生产要求：持久化会话哈希而非原 token；反代 HTTPS；备份数据库；设置最小权限；补充分布式限速。修改密码失败前不得覆盖 wrapped key。
