# 规格：双后端零知识密码库

## 目标
共享中文移动优先前端，同时运行于 Cloudflare Worker+D1 与 Linux Node+SQLite。数据不自动同步。服务端仅保存账户认证材料、被包装的库密钥和条目密文。

## 类型与验收
- account：平台、登录网址、1–20 组账号/密码、备注、标签。解密后的规范结构为 `credentials:[{username,password}]`；继续接受旧版顶层 `username/password`，首次解密或编辑时在浏览器内规范化。每组账号最长 256、密码最长 4096；拒绝畸形行、未知敏感字段及混合新旧结构。服务端 envelope 与零知识边界不变。
- website：名称、网址、说明、标签；界面与数据校验均拒绝账号/密码字段。
- note：标题、任意正文、标签。
- 类型创建后不可修改；当前分类内浏览器端模糊搜索，不向服务端发送明文查询，且仅匹配该类型可见字段。列表每条右侧提供独立“更多操作”按钮，轻量菜单含“编辑”“删除”，点击外部或 Escape 关闭且不得触发详情；详情标题栏也提供删除入口。删除前须二次确认并显示条目标题，执行中禁用且显示“删除中…”，失败给出中文提示；编辑页危险区删除保留为兜底。
- 桌面列表+详情，手机卡片+全屏详情；唯一“+ 新建”先选类型。
- 账号新建/编辑默认至少一行，提供清晰的“+ 添加账号”，按“账号 N/密码 N”标注并允许移除但始终保留至少一行；详情逐组显示并独立复制。列表/搜索绝不显示或匹配密码；搜索匹配任意账号行。320px 与 iPhone Safari 保存路径必须可用。
- 快速删除在桌面、手机与 320px 宽度可用，并具备键盘焦点、ARIA 菜单/确认对话框语义。
- 分组：账号、网站、笔记、附件各自拥有独立的客户端分组；每类有仅存在于界面状态、不写入注册表或条目的隐式 `全部` 视图；`默认` 为隐式分组，旧条目及未设置 `groupId` 的条目归入默认，默认分组不可重命名或删除。自定义分组注册表使用保留的 `settings` 加密 envelope 持久化，服务端及网络/存储不得出现分组名称或条目分配明文；空分组必须可持久化，旧客户端须能安全忽略该类型。注册表按四种顶层类型分别保存稳定随机 ID 与名称。
- 分组交互：搜索框旁提供紧凑、可访问的文件夹图标（称“分组”，避免与顶层“分类”混淆），打开适配 320px 的面板，显示默认、自定义分组及可选计数，并支持创建、重命名、删除；删除须确认并把该组条目重分配到默认。每个顶层类型独立记忆当前分组。当前分组过滤与浏览器端模糊搜索组合，均不发送明文。
- 编辑器：账号、网站、笔记的新建/编辑均含当前类型分组选择器，未选即默认；附件上传与元数据编辑含独立附件分组，同时保留图片/视频/其他文件类别过滤器且两者组合生效。
- 兼容与备份：加密导入/导出包含 `settings` envelope；旧备份无注册表时使用空注册表，旧条目规范化到默认。注册表与条目分配只在解密后校验，未知/失效 group ID 回退默认。
- 发布门禁：以 RED→GREEN 覆盖注册表契约、加密持久化、旧备份导入导出、四类型独立、空组、创建/改名/重分配/删除回默认、模糊搜索组合过滤、Chromium/WebKit 320px、网络与存储无分组明文；移动端附件工具栏须在 Chromium/WebKit 320/390px 保持搜索宽度至少 150px、完整显示默认分组、附件分类筛选至少 44px 高且独立成行，并且无重叠或横向溢出；仅在完整 `npm test`、lint、typecheck、build、双目标生产冒烟与清理全部通过后发布。任一门禁失败不得发布。
- 菜单：导入/导出、修改密码、退出。
- WebCrypto PBKDF2-SHA-256(310000)+AES-GCM；随机 vault key 被 KEK 包装；修改密码只重包密钥。
- 注册、登录/会话、改密与备份中的 `kdf` 为 `{salt,iterations:310000,hash?:'SHA-256'}`，`wrappedKey` 为 `{iv,ciphertext}`；数据库文本列只存 JSON 文本，API 始终输出对象。
- 会话 Cookie HttpOnly/Secure/SameSite=Strict、CSRF、限速、同源检查。
- 用户名存储和查询使用 `trim()` 后的原值，允许中文、空格、emoji 与常见符号；不得改变大小写或隐式 Unicode 规范化。trim 后须非空，按 JavaScript UTF-16 `String.length` 最多 80 个字符，拒绝控制/格式字符；精确匹配且唯一，违规返回 `invalid_username`。

## 命令
`npm test`、`npm run lint`、`npm run typecheck`、`npm run build`、`npm start`。

## 边界
始终校验输入与所有权；修改契约先更新文档。绝不记录密文正文、密码、密钥；绝不部署或修改旧系统。
